# Локальная многоагентная ИИ-система

Локальное Tkinter-приложение для пошаговой многоагентной симуляции с языковыми моделями через Ollama. Система рассчитана на работу на локальном компьютере без обязательных облачных API.

## Возможности

- Русскоязычный графический интерфейс с более выразительным тёмным оформлением, карточками, эмодзи-иконками, анимированной шапкой и живым индикатором статуса Ollama/симуляции.
- Контекстное меню и горячие клавиши копирования/вставки/вырезания/выделения во всех основных текстовых полях: `Ctrl+C`, `Ctrl+V`, `Ctrl+X`, `Ctrl+A` (`Command` на macOS тоже поддерживается).
- Меню **Меню → Настройки Ollama** с полями **API Ollama**, **API ключ** и **Путь к Ollama**.
- Кнопка **🚀 Запустить Ollama**, которая пытается выполнить `ollama serve` через указанный путь или через команду `ollama` из `PATH`.
- Создание, изменение и удаление агентов, краткосрочная и долгосрочная память, инструменты, статус и изолированная логическая идентичность.
- Центральная шина сообщений с прямыми и широковещательными сообщениями.
- Пошаговый движок симуляции, который обращается к выбранной модели Ollama и безопасно применяет структурированные JSON-действия.
- Проверка статуса Ollama, обновление списка моделей, выбор модели и кнопка установки, открывающая официальную страницу загрузки Ollama с локальными инструкциями.
- Постоянное состояние мира в `data/world_state.json`, включая выбранную модель, адрес Ollama API, API ключ и путь к Ollama, с восстановлением при запуске.
- Контролируемые инструменты для чтения и записи файлов внутри `workspace/`, локальных Python-фрагментов, обновления памяти и сообщений между агентами.
- Защитная обработка некорректных ответов модели: ошибки попадают в журнал, а симуляция продолжает работать.

## Запуск приложения

```bash
python main.py
```

## Как правильно подключить Ollama

### 1. Установите Ollama

Скачайте Ollama с официальной страницы: <https://ollama.com/download>.

После установки проверьте в терминале:

```bash
ollama --version
```

Если команда не найдена, значит `ollama` не добавлена в `PATH`. В этом случае в приложении откройте **Меню → Настройки Ollama** и укажите полный путь к исполняемому файлу Ollama.

Примеры путей:

- Windows: `C:\Users\<имя>\AppData\Local\Programs\Ollama\ollama.exe`
- macOS/Linux: `/usr/local/bin/ollama` или `/usr/bin/ollama`

### 2. Запустите сервер Ollama

Вариант из приложения:

1. Откройте **Меню → Настройки Ollama**.
2. Укажите **Путь к Ollama**, если команда `ollama` не работает из терминала.
3. Нажмите **🚀 Запустить Ollama**.
4. Через пару секунд нажмите **🔍 Проверить** или **↻ Модели**.

Вариант из терминала:

```bash
ollama serve
```

Если Ollama уже запущена как приложение или служба, второй запуск может быть не нужен.

### 3. Загрузите модель

Модели не появятся в выпадающем списке, пока вы не загрузите хотя бы одну модель:

```bash
ollama pull llama3.1
```

После загрузки нажмите в приложении **↻ Модели**.

### 4. Проверьте настройки API

В **Меню → Настройки Ollama** доступны три поля:

- **API Ollama** — адрес HTTP API. Для локальной Ollama обычно нужен `http://127.0.0.1:11434`.
- **API ключ** — обычно пустой для локальной Ollama. Заполняйте только если вы подключаетесь к удалённому Ollama API или прокси, который требует `Authorization: Bearer <ключ>`. Ключ сохраняется локально в `data/world_state.json`, поэтому не публикуйте этот файл.
- **Путь к Ollama** — путь к `ollama`/`ollama.exe`, чтобы кнопка **🚀 Запустить Ollama** могла запустить сервер.

Если Ollama «не появляется», чаще всего причина одна из этих:

1. Сервер Ollama не запущен — нажмите **🚀 Запустить Ollama** или выполните `ollama serve`.
2. Не загружена модель — выполните `ollama pull llama3.1`.
3. Неверный API адрес — проверьте, что указан `http://127.0.0.1:11434` или ваш реальный удалённый адрес.
4. Неверный путь к Ollama — укажите полный путь к исполняемому файлу в настройках.
5. Удалённый API требует ключ — вставьте ключ в поле **API ключ**.

## Формат ответа агента

Агенты должны возвращать строгий JSON:

```json
{
  "thought": "краткое приватное резюме рассуждения",
  "actions": [
    {"tool": "remember", "args": {"content": "важное событие"}}
  ],
  "messages": [
    {"recipient": "broadcast", "content": "привет, агенты"}
  ]
}
```
